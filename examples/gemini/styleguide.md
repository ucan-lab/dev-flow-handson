# チームレビュー観点 (for Gemini Code Assist)

この文書は Gemini Code Assist がレビュー時に参照するチーム規約です。
人間向けに読めるマークダウンで書けば Gemini が解釈します。

## 優先度の高い観点

1. **セキュリティ**
   - ハードコードされた秘密情報 (API キー / パスワード / トークン) がないか
   - ユーザー入力のサニタイズ / パラメータ化クエリ
   - 認可チェック漏れ
2. **可読性**
   - 関数 50 行以内 / ファイル 800 行以内 / ネスト 4 段以内
   - マジックナンバーは定数化されているか
3. **エラーハンドリング**
   - 例外を握りつぶしていないか
   - 境界 (外部 API / ユーザー入力) でのバリデーション
4. **テスト**
   - 新機能に対するテストが追加されているか
   - テストが実装と同じ PR に含まれているか

## 指摘のトーン

- 断定ではなく提案ベース ("〜した方が良いかもしれません")
- 必ず理由を添える ("なぜならこのパスは外部から呼ばれるため...")
- 小さな nitpick は `nit:` プレフィックス

## 対象外

- テストファイル内のマジックナンバー
- 自動生成コード (`*_pb.go`, `*.gen.ts` など)
- ドキュメントファイル (タイポレベルは無視)
