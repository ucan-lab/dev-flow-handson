## このハンズオンの目的

実装が終わった後の開発フローを、AI で効率化する方法を実際に手を動かして学びます。

対象は `diff 確認 -> コミット -> ローカル AI レビュー -> PR 作成 -> PR レビュー -> レビュー対応` です。コミット粒度、Conventional Commits、PR テンプレート、レビュー観点などのルールを AI に守らせ、人間は仕様判断や設計判断に集中できる状態を目指します。

AI の実行をただ待っているだけでは、大きな効率化にはなりません。AI に任せる作業と人間が進める作業を分け、待ち時間を別タスクに回せるワークフローを作ります。

今回紹介する Claude Skills は完成品としてそのまま使うものではなく、自分やプロジェクトのルールに合わせて調整するためのひな形です。
